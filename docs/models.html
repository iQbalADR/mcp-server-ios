Overview

These Modelfiles create specialized Ollama models with expert iOS/Swift knowledge baked into their system prompts. Each model is optimized for specific tasks.

# Create all models at once
./scripts/create_models.sh

# Or create the recommended model individually
ollama create ios-qwen-coder -f models/Modelfile.qwen-coder

iOS Qwen Coder ‚≠ê Recommended

ios-qwen-coder

Recommended Code-Optimized 16K Context

Built on Qwen2.5-Coder-7B, a model trained specifically on code. Delivers significantly better code generation, completion, and understanding than general-purpose LLMs. 16K context window for handling large codebases.

Why Qwen2.5-Coder?

  • Trained specifically on code datasets
  • 16K context window (2x larger)
  • Better code completion accuracy
  • Smarter refactoring suggestions
  • More precise Swift syntax
  • Reduced hallucination on APIs

‚ö° Agentic Capabilities

When used with this server, this model can:

  • Explore project files autonomously
  • Read and analyze code context
  • Create and edit files directly
  • Execute commands to build/test

Same Expertise as iOS Architect

  • Swift 5.9+ (async/await, actors, macros)
  • UIKit & SwiftUI
  • Architecture Patterns (MVVM, VIPER, TCA)
  • Protocol-Oriented Programming
  • Memory Management
  • Full iOS SDK Knowledge

Create Command

ollama create ios-qwen-coder -f models/Modelfile.qwen-coder

Test It

ollama run ios-qwen-coder "Create a Swift networking layer with async/await and Codable"

Creating Models

Quick Setup

# Make script executable
chmod +x scripts/create_models.sh

# Create all models
./scripts/create_models.sh

Manual Creation

# 1. Pull base models
ollama pull qwen2.5-coder:7b
ollama pull llama3.1:8b

# 2. Create each model
ollama create ios-qwen-coder -f models/Modelfile.qwen-coder
ollama create ios-swift-architect -f models/Modelfile.ios-architect
ollama create swiftui-specialist -f models/Modelfile.swiftui
ollama create ios-code-reviewer -f models/Modelfile.reviewer

# 3. Verify
ollama list

Using with MCP Server

Update config.yaml:

ollama:
  chat_model: "ios-qwen-coder"  # Recommended
  embedding_model: "nomic-embed-text:v1.5"

Customization

Model Parameters

Parameter Default Description
temperature 0.1 Lower = more deterministic code
repeat_penalty 1.1 Reduces repetition in output
num_ctx 16384 Context window size (Qwen supports up to 128K)
top_p 0.9 Nucleus sampling threshold
num_predict 4096 Max tokens to generate

Alternative Base Models

Change the FROM line in any Modelfile:

# Best for code (recommended)
FROM qwen2.5-coder:7b

# Good general purpose
FROM llama3.1:8b

# Smaller, faster
FROM codellama:7b

# Larger, better quality
FROM codellama:13b

# Alternative code model
FROM deepseek-coder

Creating Your Own Model

# 1. Create a new Modelfile
touch models/Modelfile.my-model

# 2. Add content
cat << 'EOF' > models/Modelfile.my-model
FROM llama3.1:8b

SYSTEM """Your custom expertise description..."""

PARAMETER temperature 0.1
PARAMETER num_ctx 8192
EOF

# 3. Create the model
ollama create my-custom-model -f models/Modelfile.my-model

# 4. Test it
ollama run my-custom-model
